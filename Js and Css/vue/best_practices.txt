########################################## ПУТИ КОМПОНЕНТОВ ПРИ ИМПОРТЕ:

в путях компонентов лучше указывать @, а не ./ так как при переносе компонента в другой адрес 
возникнут проблемы



########################################## КЛЮЧИ НА V-FOR:

не рекомендуется использовать индексы как ключи - так как при удалении элементов из списков,
индексы могут сдвигаться

суть - для алгоритмов нужен уникальный ключ, по которому можно будет одноначно найти
Node, созданный из элемента списка в вирьуальном DOM



########################################## PROPS:

пропсы в дочернем компоненте изменять нельзя, данные должны как-то 
изменяться в родителе, а дочерний компонент должен получать их 
в уже измененном виде: сам он их напрямую не изменяет

пропсы вообще лучше не изменять



########################################## VUEX:

VUEX нужен для того чтобы данные и логику по работе с этими данными
выносить из компонента
- компонент это отображение vue
- а сами данные и логика по работе с этими данными выносится в store

1) получать информацию не напрямую из state, а через getters
2) изменять состояние напрямую мы не можем, но можем делать это с помощью мутаций
- мутации это функции, внутри которых мы меняем значения поля в состоянии
3) вызываем мутации через actions
4) асинхронно делаем только actions
5) состояние рекомендуется объявлять в виде функции, которая возвращает объект
