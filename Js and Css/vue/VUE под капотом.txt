виртуальный DOM это концепт в котором идеальный или "виртуальный" DOM - представление UI - хранится
в памяти и синхронизирован в реальным DOM. Концепт был унаследован из React.

виртуальный DOM это паттерн, а не технология, поэтому вот вам объяснение его сути через пример.

const vnode = {
  type: 'div',
  props: {
    id: 'hello'
  },
  children: [
    /* more vnodes */
  ]
}

Здесь переменная vnode - это просто JS object, which representing a <div> element.
Он содержит всю информацию, которая нам нужна для создания реального элемента.
Также переменная содержит дочерние vnodes, которые составляют корень дерева
виртуального DOM.

A runtime renderer can walk a virtual DOM tree and construct a real DOM tree from it. 
This process is called mount.

If we have two copies of virtual DOM trees, the renderer can also walk 
and compare the two trees, figuring out the differences, and apply 
those changes to the actual DOM. This process is called patch, 
also known as "diffing" or "reconciliation".

Главное преимущество virtual DOM состоит в том, что он дает разработчику возможность 
делать с UI все что хочешь, при этом оставляя DOM манипуляцию рендеререру.

В общем - virtual DOM все конкретно упрощает.
