при установлении пакета через npm i package_name
в корне пакета также есть package.json с его подробным описанием и инструкциями
dist - в этой директории находятся конечные скомпилированные файлы

некоторые пакеты содержат и исполняемые .bin файлы (например файл для запуска тестов, линтер)
исполняемые файлы всех исполняемых пакетов выносятся в корневую директорию node modules откуда мы можем с ними работать
когда мы устанавливаем пакет глобально, bin файл пакета становится доступен прямо из консоли

пакеты можно устанавливать списком через пробел
npm i gulp mocha

node_modules

при установки любого пакета мы получаем последнюю рекомендованную версию

версии пакета, три числа 1.0.0 
1.0.0 (версия patch - выход нового patch означает что в пакет были внесены незначительные изменения) можно обновлять
1.2 (версия minor - в пакет были внесены серьезные изменения, например добавлен новый функционал, но при этом
осталась совместимость с прошлой минорной версией) .0 можно обновлять
2 .0.0 (версия major - в пакет были внесены критические изменения и он больше не совместим с прошлой версией) - такое
обновление нельзя ставить неглядя, так как код может поломаться

!!! !!! !!!
	"bootstrap": "^4.1.1"
символ каретки ^ то версия будет зафиксирована и не будет обновлятся при npm up

	"bootstrap": "~4.1.1"
если мы хотим разрешить только обновление patch то перед версией нужно поставить ~ тильду

	"bootstrap": "*"
если мы хотим обновить мажорную версию, чего не стоит делать в production, то надо указать символ звездочки

npm list - увидеть какие пакеты установлены локально
npm list --depth=0 сделать тоже самое с ограниченной глубиной
npm-list -g --depth=0 список пакетов установленных глобально

#################
npm outdated проверяет установленные в проекте пакеты на наличие обновлений

#################
Удаление пакета
npm remove bootstrap
npm uninstall bootstrap
npm un bootstrap

#################
Установка пакета определенной версии
npm i bootstrap@4.1.1

при этом можно установить пакет таким образом
npm i bootstrap@4.1
npm i bootstrap@4
тогда patch или major установит последний из доступных

#################
обновление пакета происходит так
npm update
npm up
обновляет все пакеты
